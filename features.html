EvilGoy

Profiles Tracking Fingerprinting Architecture

Feature Overview

EvilGoy is a local-only privacy extension designed to reduce tracking, fingerprinting, and data leakage without breaking websites.

Nothing leaves your browser. No servers. No analytics. No accounts.

Profiles System

Default

  • Balanced privacy protection
  • Minimal website breakage
  • Recommended for daily browsing

Strict

  • Maximum privacy
  • All defenses enabled
  • Some sites may break (logins, video calls)

Off

  • Disables EvilGoy completely
  • Full site compatibility
  • Ideal for banking or work tools

Tracking & Leak Prevention

Tracker Blocker

  • Blocks Google Analytics & DoubleClick
  • Blocks Meta (Facebook) pixels
  • Blocks TikTok tracking scripts
  • Blocks generic ad & telemetry endpoints
  • Uses Chrome declarativeNetRequest
  • No request inspection or logging

Anti-WebRTC Leak

  • Removes WebRTC APIs entirely
  • Prevents real IP leaks even with VPNs
  • Stops WebRTC-based fingerprinting

Referrer Trimming

  • Removes referrer headers
  • Prevents link-based tracking
  • Stops referral analytics & correlation

Fingerprint Defense

User-Agent Spoofing

  • Replaces real browser & OS details
  • Prevents OS detection
  • Reduces targeted exploit risk

Timezone Normalization

  • Forces UTC timezone
  • Removes geographic inference
  • Blocks region-based fingerprinting

Screen Size Spoofing

  • Returns standardized screen dimensions
  • Prevents laptop & monitor fingerprinting
  • Stops multi-monitor detection

Canvas Fingerprint Protection

  • Adds subtle pixel noise
  • Breaks deterministic canvas hashes
  • Makes fingerprint IDs unreliable

Optional & Advanced Features

Cookie Hardening

  • Prevents persistent cookies
  • Allows session-only behavior
  • Disabled by default for compatibility

Search Query Poisoning

  • Injects periodic fake search queries
  • Confuses interest profiling algorithms
  • Disabled by default due to aggressiveness

Privacy-Safe Request Inspector

  • Shows number of blocked requests
  • No URLs, domains, or request data
  • No logging, ever

Modular Architecture

  • Each feature is isolated
  • No hidden behavior
  • Safe future expansion

Local-Only Storage

  • Uses chrome.storage.local
  • No sync, no cloud
  • No telemetry or analytics

Chrome Web Store Safe

  • Manifest V3 compliant
  • No remote code execution
  • Fully auditable
